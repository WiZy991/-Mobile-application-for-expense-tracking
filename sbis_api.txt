2. Регистрация:
Для организации (ООО) - https://online.sbis.ru/service/?srv=1
Для ИП - https://online.sbis.ru/service/?srv=1

При регистрации когда пользователь вводит в поле ИНН свой ИНН, то делается запрос в сбис на информацию о пользователе по этому ИНН
и автоматически должны заполниться поля в профиле по той информации что придет со сбиса по пользовательскому ИНН.

Команда «СБИС.ИнформацияОКонтрагенте» возвращает информацию о контрагенте по ИНН и КПП — например, можно узнать идентификатор участника документооборота. Если по переданным параметрам карточка контрагента не найдена, будет создана новая.

"СвЮЛ": объект:
"ИНН": строка, номер состоит из 10 символов для российских юридических лиц. Для иностранных организаций номер определяется требованиями законодательства их стран
"КПП": строка, 9 символов
"КодФилиала": строка
"КодСтраны": строка, код страны по стандарту ISO 3166-1 numeric
"Название": строка, название организации
"АдресЮридический": строка, юридический адрес головной организации
"Идентификатор": массив объектов, содержит список идентификаторов абонентских ящиков:
"ИдентификаторУчастника": строка, идентификатор участника документооборота
"Комментарий": строка, комментарий к идентификатору участника документооборота
"Оператор": объект, описание оператора ЭДО, выдавшего идентификатор участника документооборота
"Идентификатор": строка, код оператора ЭДО из трех символов
"Название": строка, название оператора ЭДО
"Основной": строка, используется при отправке документов по умолчанию. Возможные значения «Да» или «Нет»
"Роуминг": строка, признак, что у оператора ЭДО роуминг. Возможные значения «Да» или «Нет»
"СостояниеПодключения": объект
"Код": строка, код состояния подключения. 
Возможные значения:
«0» — «Подключен» — контрагент подключен к ЭДО или роумингу;
«1» — «Отключен» — контрагент отключен от документооборота. Отправка первичных документов невозможна; вторичных, например ответных титулов, извещений — возможна;
«2» — «Не подключен» — контрагент не имеет подключения к ЭДО. При отправке документов в этом состоянии будет отправлено приглашение к участию в документообороте, документы будут ожидать подключения контрагента.
"Описание": строка, описание состояния подключения
Комментарий
Если передан объект «Ответственный», то при создании нового контрагента он устанавливается в качестве ответственного.
Идентификатор участника документооборота может меняться в процессе работы. Не рекомендуется сохранять и использовать этот идентификатор.
Обработка значения поля «Идентификатор»:
если передан полный идентификатор участника документооборота, он считается приоритетным для определения контрагента, прочие реквизиты игнорируются. Saby находит контрагента по идентификатору и возвращает полные реквизиты, за исключением «АдресЮридический». Если заказан расширенный ответ метода, возвращает список идентификаторов. При этом запись в «Компании/Наши контрагенты» не создается;
если передан трехбуквенный префикс оператора ЭДО, то при поиске контрагента Saby получает все его идентификаторы и возвращает в результат только те, у которых есть переданный префикс. Кроме этого возвращает остальные реквизиты — ИНН, КПП, Название, КодСтраны и другие.
Если в качестве идентификатора вернулась пустая строка или пустая выборка при выводе расширенной информации, значит, контрагент отказался от участия в электронном документообороте. Документы, которые вы попытаетесь ему отправить, не будут доставлены и попадут в реестр ошибок.
Если по переданным реквизитам (кроме «Идентификатор») контрагент не найден, Saby создаст его в разделе «Компании/Наши контрагенты» и заполнит реквизиты по данным сервиса проверки партнеров.
Обработка значения поля «ДопПоля»:
«СписокИдентификаторов» — в результате метода в поле «Идентификатор» вернется список подключенных идентификаторов с расширенной информацией. Если контрагент не подключен — один виртуальный идентификатор. Если передан трехсимвольный идентификатор оператора в «Участник.Идентификатор», список идентификаторов будет отфильтрован по нему;
«СФилиалами» — вернется идентификатор участника документооборота головной организации и филиалов, которые совпадают по ИНН и КПП;
значение не передано — вернется основной идентификатор участника документооборота.

4. СБИС.СписокСотрудников
Команда возвращает информацию о списке сотрудников.

Параметры запроса 
Адрес запроса: https://online.sbis.ru/service/?srv=1

"Параметр":
"Фильтр": объект:
"НашаОрганизация": объект:
"СвЮЛ": объект:
"ИНН": строка, ИНН. Для юридических лиц в РФ номер состоит из 10 символов. Для иностранных организаций номер определяет законодательство их стран
"КПП": строка, 9 символов
"СвФЛ": объект:
"ИНН": строка, ИНН. Для индивидуальных предпринимателей и физических лиц в РФ номер состоит из 12 символов. Для иностранных организаций номер определяет законодательство их стран
"ВернутьУволенных": строка. Возможные значения:
«Да» — вернется список сотрудников, которые уже уволены или будут приняты на работу после сегодняшнего дня;

Чтобы добавить вложения к документу, укажите их в «Этап.Вложения». При локальном подписании можно приложить электронные подписи.
При отправке через оператора ЭДО действует ограничение на число вложений по тарифу роумингу. Если для организации установлены разные лимиты, применяется наиболее жесткое ограничение.
При выполнении команды «СБИС.ВыполнитьДействие» происходит:

предварительный вызов «СБИС.ПодготовитьДействие»;
передача информации о сертификате для подписания ЭП (объект «Этап.Действие.Сертификат»);
передача файла подписи (объект «Этап.Вложение.Подпись.Файл.ДвоичныеДанные»).
После вызова «СБИС.ПодготовитьДействие» вы можете заменить сгенерированные сервером служебные вложения на свои. При этом укажите идентификаторы заменяемых вложений и передайте двоичные данные.

На этапах внешнего документооборота выполняется форматно-логический контроль пользовательских файлов.

В запросе команды должно быть заполнено поле «Документ.Идентификатор» или «Документ.Редакция.Идентификатор». При заполненном «Документ.Идентификатор» обрабатывается этап и действия последней редакции. Если заполнено поле «Документ.Редакция.Идентификатор», то обрабатывается этап и действия конкретной редакции.

В команде мы рекомендуем передавать идентификатор этапа и идентификатор редакции.

Не используйте адресацию по названию этапов. Это может привести к ошибкам:

будет подготовлена неверная редакция документа — при адресации по идентификатору документа на момент вызова метода может появиться новая редакция;
может быть создан одноименный этап — если одноименные этапы были переназначены текущему или другому пользователю.
Если для подписания документа используется ЭП на носителе, исключите отправку прочих входящих в формат полей, за исключением поля «Подпись».

Если для подписания используется отложенный сертификат, передайте запрос в поле «Сертификат.Ключ.Тип» и укажите тип сертификата — «Отложенный» или «ОтложенныйСПодтверждением». Реквизиты сертификата не запоминаются. Этап будет обработан владельцем сертификата с использованием любого из доступных ему собственных или доверенных сертификатов, которые подходят под требования регламента.
Если функция «СБИС.ВыполнитьДействие» вызывается не для этапа подготовки к отправке документа, передайте в команде значение поля «Документ.Этап.Идентификатор». Оно может быть получено от вызова другой команды, например «СБИС.ПодготовитьДействие».
В крайнем случае можно указать «Документ.Этап.Название», но этот способ ненадежен. Он допускает подготовку к неверному этапу:

при отправке новой редакции документа;
попытке подготовки этапа по предыдущей редакции.
Чтобы обработать и отправить отдельные файлы, в объекте «Документ» укажите:
в поле «ДопПоля» — «СложноеУтверждение»;
в поле «Вложение» — идентификаторы обрабатываемых файлов.

Чтобы запустить задачу в документооборот, выполните запрос «СБИС.ВыполнитьДействие». В запросе укажите:

«Этап.Название»=«Выполнение» для запуска ДО. Если в регламенте один стартовый переход, то действие можно не передавать. Условные переходы можно называть одинаково — действие сработает на тот переход, который подошел под условие;

7. СБИС.ИнформацияОКонтрагенте
Команда «СБИС.ИнформацияОКонтрагенте» возвращает информацию о контрагенте по ИНН и КПП — например, можно узнать идентификатор участника документооборота. Если по переданным параметрам карточка контрагента не найдена, будет создана новая.
Адрес запроса: https://online.sbis.ru/service/?srv=1
Для организации
Для ИП
Для физического лица
Параметры запроса
* — обязательный параметр.
Синий — только для чтения.
"Участник": объект:
"Идентификатор": строка, идентификатор участника документооборота либо трехсимвольный префикс идентификатора оператора ЭДО
"СвФЛ": объект:
"ЧастноеЛицо": строка со значением «Нет»
"ИНН"*: строка, 12 символов для российских индивидуальных предпринимателей и физических лиц. Для иностранных организаций номер определяется требованиями законодательства их стран
"Фамилия"*: строка
"Имя"*: строка
"Отчество": строка
"КодФилиала": строка
"GLN": строка, международный идентификатор организации
"Ответственный": объект:
"Фамилия": строка
"Имя": строка
"Отчество": строка
"Идентификатор": строка, идентификатор во внешней системе
"Подразделение": объект:
"Название": строка, название подразделения
"Идентификатор": строка, код подразделения
"ДопПоля": строка. Возможные значения: «СписокИдентификаторов» и «СФилиалами». Чтобы передать несколько значений, укажите их через запятую без пробела
Результат
"СвФЛ": объект:
"ЧастноеЛицо": строка
"ИНН": строка, 12 символов для российских индивидуальных предпринимателей и физических лиц. Для иностранных организаций номер определяется требованиями законодательства их стран
"Фамилия": строка
"Имя": строка
"Отчество": строка
"КодФилиала": строка
"Идентификатор": строка, содержит идентификатор участника документооборота (на момент запроса). Пустая строка, если контрагент отказался от участия в документообороте
Комментарий
Если передан объект «Ответственный», то при создании нового контрагента он устанавливается в качестве ответственного.
Если параметр «ЧастноеЛицо» не передан, Saby ищет сведения о контрагенте в доверенных источниках. Если информация найдена и контрагент окажется индивидуальным предпринимателем, то он будет создан как ИП. Иначе — как физическое лицо.
Идентификатор участника документооборота может меняться в процессе работы. Не рекомендуется сохранять и использовать этот идентификатор.
Обработка значения поля «Идентификатор»:
если передан полный идентификатор участника документооборота, он считается приоритетным для определения контрагента, прочие реквизиты игнорируются. Saby находит контрагента по идентификатору и возвращает полные реквизиты. Если заказан расширенный ответ метода, возвращает список идентификаторов. При этом запись в «Компании/Наши контрагенты» не создается;
если передан трехбуквенный префикс оператора ЭДО, то при поиске контрагента Saby получает все его идентификаторы и возвращает в результат только те, у которых есть переданный префикс. Кроме этого возвращает остальные реквизиты — ИНН, КПП, Название, КодСтраны и другие.
Если в качестве идентификатора вернулась пустая строка или пустая выборка при выводе расширенной информации, значит, контрагент отказался от участия в электронном документообороте. Документы, которые вы попытаетесь ему отправить, не будут доставлены и попадут в реестр ошибок.
Если по переданным реквизитам (кроме «Идентификатор») контрагент не найден, Saby создаст его в разделе «Компании/Наши контрагенты» и заполнит реквизиты по данным сервиса проверки партнеров.
Обработка значения поля «ДопПоля»:
«СписокИдентификаторов» — в результате метода в поле «Идентификатор» вернется список подключенных идентификаторов с расширенной информацией. Если контрагент не подключен — один виртуальный идентификатор. Если передан трехсимвольный идентификатор оператора в «Участник.Идентификатор», список идентификаторов будет отфильтрован по нему;
«СФилиалами» — вернется идентификатор участника ДО головной организации и филиалов, которые совпадают по ИНН и КПП;
значение не передано — вернется основной идентификатор участника ДО.

3 — снята с регистрации
4 — ожидание активации
6 — нет лицензии

14.Contractor.Find
Команда «Contractor.Find» возвращает результат полнотекстового поиска по переданным реквизитам. Поиск происходит по полям «ИНН», «ОГРН», «Название», «Адрес», «Телефон», «e-mail», «web-сайт». В поиске участвуют только головные контрагенты, по филиалам поиск не производится.

Адрес запроса: https://api.saby.ru/spp-rest-api/service/

Структура запроса 
Параметр

Тип

Описание

requisites

Строка

Реквизиты искомого контрагента

page

Число

Номер страницы, нумерация начинается с 0

size

Число

Размер страницы

Схема ответа 
Массив в формате JSON. Поля массива представлены в таблице.

Поле

Тип

Описание

INN

Строка

ИНН организации или ИП

KPP

Строка

КПП контрагента

Name

Строка

Сокращенное название

OGRN

Строка

ОГРН контрагента

State

Строка

Перечисляемое значение. Состояние организации:

0 — ликвидированное;
1 — действующее;
2 — в состоянии реорганизации;
3 — не определено;
4 — в состоянии ликвидации.

   "method": "Contractor.Find",
   "params": {
      "requisites": "7724240754",
      "page": 0,
      "size": 20
   },
   "id": 1
}
Пример ответа

      "INN": "7724240754",

   }
}


15. Получить список товаров по API для салонов
Запрос возвращает информацию о товарах и услугах:
из прайс-листа или колонки цен — если передан идентификатор в параметре priceListId;
из каталога — если идентификатор не указан.

Адрес запроса: https://api.sbis.ru/retail/v2/nomenclature/list?

26. Двухфакторная аутентификация
Если у пользователя настроена двухфакторная аутентификация, методы аутентификации вернут ошибку вида JSON с кодом 303. В структуре dump будет содержаться информация для подтверждения входа. Далее надо:

Вызвать метод AuthSendCode, передав значения полей «ResourceID» и «SessionID» из dump в качестве параметров. Метод сформирует код и отправит его на подтвержденный мобильный телефон пользователя.
Вызовете метод AuthConfirmCode. В качестве параметров передайте:
код подтверждения из полученного смс-сообщения,
временный идентификатор сессии(«ResourceID»), который вернулся в результате AuthSendCode,
значение поля «MethodToValidate» из dump.
В случае успеха метод вернет идентификатор сессии.
Пример ошибки 
{
  'code': 303,
  'detail': 'Требуется подтверждение действия',
  'dump': {
    'MethodToValidate': 'САП.AuthenticateExPart2',
    'ResourceID': '3a23a986-3b0f-4763-89a5-519f7da8a271',
    'SessionID': '00610413-006104f7-eeba-fbd5a09730f7eaba'
  },
  ...
}
Пример кода для AuthSendCode 
    query_id = str(uuid.uuid4())
    module_method = "ExtSdk2.AuthSendCode"
    parameters_module_method = json.dumps({"ResourceID": "3a23a986-3b0f-4763-89a5-519f7da8a271", "SessionID": 00610413-006104f7-eeba-fbd5a09730f7eaba}, ensure_ascii=True)
    host = online.sbis.ru
    ole.CallMethodWithoutAuth( query_id, guid_module, module_method, parameters_module_method, host )
    json_string = ole.ReadAllObject()

import json
import requests
def login(user_name, pwd) -> str:
    """
    Авторизация
    """
    return invoke(
        "СБИС.Аутентифицировать",
        {"Параметр": {"Логин": user_name, "Пароль": pwd}},
        None,
    )["result"]
def invoke(method: str, params: dict, auth_sid) -> dict:
    """
    Вызов метода
    """
    payload = {
        "jsonrpc": "2.0",
        "method": method,
        "params": params,
        "protocol": 2,
        "id": 0,
    }
    headers = {
        "Host": "online.sbis.ru",
        "Content-Type": "application/json-rpc; charset=utf-8",
        "Accept": "application/json-rpc",
    }
    if auth_sid:
        headers["X-SBISSessionID"] = auth_sid
        url = "https://online.sbis.ru/service/"
    else:
        url = "https://online.sbis.ru/auth/service/"
    r = requests.post(url, headers=headers, data=json.dumps(payload))
    return json.loads(r.text)
