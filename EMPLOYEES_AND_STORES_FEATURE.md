# Функционал сотрудников и магазинов

## Что реализовано

### 1. База данных

#### Таблица `stores` (магазины)
- `id` - ID магазина
- `client_id` - ID директора/компании
- `name` - Название магазина
- `address` - Адрес магазина
- `phone` - Телефон магазина (опционально)
- `is_active` - Активен ли магазин
- `created_at`, `updated_at` - Временные метки

#### Таблица `employees` (сотрудники)
- `id` - ID сотрудника
- `client_id` - ID директора/компании
- `store_id` - ID магазина (может быть NULL)
- `phone` - Номер телефона (уникальный, используется для входа)
- `name` - Имя сотрудника
- `role` - Роль сотрудника ('employee', 'manager')
- `is_active` - Активен ли сотрудник
- `last_login_at` - Время последнего входа
- `created_at`, `updated_at` - Временные метки

#### Обновление таблицы `clients`
- `role` - Роль пользователя ('director', 'employee')
- `parent_client_id` - ID директора (для сотрудников)

### 2. API Endpoints

#### Магазины (`/api/stores`)
- `GET /api/stores` - Получить все магазины директора
- `POST /api/stores` - Создать новый магазин
- `PUT /api/stores/:id` - Обновить магазин
- `DELETE /api/stores/:id` - Удалить магазин

#### Сотрудники (`/api/employees`)
- `GET /api/employees` - Получить всех сотрудников директора
- `POST /api/employees` - Добавить сотрудника по номеру телефона
- `PUT /api/employees/:id` - Обновить сотрудника
- `DELETE /api/employees/:id` - Удалить сотрудника
- `POST /api/employees/auth/phone` - Авторизация сотрудника по телефону (без пароля)

### 3. Авторизация сотрудников

Сотрудники могут входить в приложение по номеру телефона без пароля:
1. Директор добавляет сотрудника через API, указывая номер телефона
2. Сотрудник входит через `POST /api/employees/auth/phone` с номером телефона
3. Система автоматически создает аккаунт клиента для сотрудника (если его еще нет)
4. Возвращается JWT токен с ролью 'employee'

### 4. Поддержка ИП

Для ИП (индивидуальных предпринимателей) реализована полная поддержка:
- Определение типа по длине ИНН (12 цифр = ИП, 10 цифр = ООО)
- Обработка ОГРНИП вместо ОГРН
- КПП не применяется для ИП (отображается "Не применяется")
- Подтягивание данных из SBIS для ИП (СвФЛ вместо СвЮЛ)
- Нормализация названий компаний (ИП вместо "ИНДИВИДУАЛЬНЫЙ ПРЕДПРИНИМАТЕЛЬ")

## Примеры использования API

### Добавить магазин
```bash
POST /api/stores
Authorization: Bearer <token>
{
  "name": "Магазин на Ленина",
  "address": "г. Владивосток, ул. Ленина, д. 1",
  "phone": "+79991234567"
}
```

### Добавить сотрудника
```bash
POST /api/employees
Authorization: Bearer <token>
{
  "phone": "+79991234568",
  "name": "Иван Иванов",
  "store_id": 1,
  "role": "employee"
}
```

### Вход сотрудника
```bash
POST /api/employees/auth/phone
{
  "phone": "+79991234568"
}
```

### Получить всех сотрудников
```bash
GET /api/employees
Authorization: Bearer <token>
```

## Безопасность

- Только директора могут создавать/редактировать магазины и сотрудников
- Сотрудники имеют ограниченный доступ (только поддержка)
- Авторизация по телефону работает только для активных сотрудников
- Все запросы требуют JWT токен (кроме входа сотрудника)
