## Архитектура упрощенного решения

```
Клиент (Mobile App)
    ↓
    sbisGetClientFromCRM(inn)
    ↓
Backend API (/crm-client-oauth)
    ↓
    Проверка сессии (обычная авторизация)
    ├─ Если нет сессии → 401 Unauthorized
    └─ Если есть сессия → продолжаем
    ↓
    Поиск контрагента в CRM:
    ├─ Навигация.Контрагент.Список
    ├─ Навигация.Контрагент.Прочитать
    └─ Контрагент.Список (fallback)
    ↓
    Получение сделок:
    ├─ CRMLead.getList
    └─ Сделка.Список (fallback)
    ↓
    Получение документов:
    ├─ Документ.СписокДокументов
    └─ Реестр.СписокЗаписей
    ↓
    Возврат результата клиенту
```

## Что изменилось в коде

### Файл: `backend/src/routes/sbisProxy.js`

**До (OAuth):**
```javascript
router.post('/crm-client-oauth', async (req, res) => {
  // Получаем OAuth токен
  let oauthToken = oauthTokens.get(userKey)
  
  // Если токена нет - выполняем OAuth авторизацию
  if (!oauthToken) {
    const authResponse = await axios.post(SBIS_OAUTH_URL, {
      jsonrpc: '2.0',
      method: 'СБИС.Аутентифицировать',
      params: {
        Приложение: SBIS_APP_CLIENT_ID,
        Пароль: SBIS_APP_SECRET,
      },
      ...
    })
    oauthToken = authResponse.data.result
  }
  
  // Используем OAuth токен
  headers: { 'X-SBISSessionID': oauthToken }
})
```

**После (обычная авторизация):**
```javascript
router.post('/crm-client-oauth', async (req, res) => {
  // Получаем обычную сессию
  let sessionId = userSessions.get(userKey)
  
  // Если сессии нет - возвращаем ошибку
  if (!sessionId) {
    return res.status(401).json({
      success: false,
      error: 'Требуется авторизация в СБИС',
      hint: 'Сначала вызовите /auth с логином и паролем',
    })
  }
  
  // Используем обычную сессию
  headers: { 'X-SBISSessionID': sessionId }
})
```

**Изменения:** ~50 строк кода упрощено

## Преимущества упрощенного подхода

### ✅ Плюсы:

1. **Проще** - не нужны OAuth credentials (app_client_id, app_secret)
2. **Понятнее** - обычная авторизация логин/пароль
3. **Меньше кода** - убрано ~50 строк OAuth логики
4. **Работает** - все эндпоинты функционируют правильно

### ⚠️ Ограничения:

1. **Права доступа** - ограничены правами конкретного пользователя
2. **Таймаут сессии** - сессия может истечь (нужна повторная авторизация)
3. **Масштабируемость** - для большого количества запросов OAuth был бы лучше

## Проверка работы в реальном приложении

### Mobile App

Процесс регистрации клиента:

1. Клиент вводит ИНН

  console.log('Сделок:', crmResult.data.deals.length);
  console.log('Документов:', crmResult.data.documents.length);
  
  setCompanyData({
    ...crmResult.data.contractor,
    deals: crmResult.data.deals,
    documents: crmResult.data.documents,
    fromCRM: true, // ← флаг для UI
  });
}

## Решение проблем

### Ошибка "Требуется авторизация"

**Причина:** Нет активной сессии СБИС

**Решение:** Выполните авторизацию:
```bash
curl -X POST http://localhost:3000/api/sbis-proxy/auth \
  -H "Content-Type: application/json" \
  -d '{"login":"YOUR_LOGIN","password":"YOUR_PASSWORD","userId":"test"}'
```

### Ошибка "Клиент не найден в CRM"

**Причины:**
1. Клиента нет в вашей CRM СБИС
2. Недостаточно прав у пользователя
3. Неправильный ИНН

**Решение:** 
- Проверьте наличие клиента в веб-интерфейсе СБИС
- Используйте ИНН из вашей CRM

### Ошибка авторизации

**Причина:** Неправильный логин/пароль

**Решение:** 
- Проверьте credentials
- Попробуйте авторизоваться на online.sbis.ru
