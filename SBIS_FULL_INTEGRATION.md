# 🔗 Полная интеграция с API СБИС

## 📋 Обзор

Реализована полная интеграция с API СБИС для автоматизации работы с клиентами, ресурсами и уведомлениями.

## ✅ Реализованные функции

### 1. Мониторинг ресурсов клиентов

#### Типы ресурсов:
- **ФН (Фискальные накопители)** - через API ОФД
- **Лицензии Эвотор** - через API СБИС
- **Лицензии Атол** - через API СБИС
- **ОФД** - через API ОФД
- **Другие лицензии и подписки**

#### Автоматические функции:
- ✅ Мониторинг сроков действия (проверка каждый день в 9:00)
- ✅ Уведомления о приближающихся сроках:
  - Срочно (7 дней до окончания)
  - Предупреждение (30 дней)
  - Информация (60 дней)
- ✅ Автоматическое продление при наличии средств (каждый день в 10:00)
- ✅ Создание счетов в СБИС при продлении
- ✅ Синхронизация данных из СБИС

### 2. API Endpoints

#### Ресурсы клиентов
- `GET /api/resources` - Получить все ресурсы клиента
- `GET /api/resources/:id` - Получить детали ресурса
- `POST /api/resources` - Добавить ресурс
- `PUT /api/resources/:id` - Обновить ресурс
- `DELETE /api/resources/:id` - Удалить ресурс
- `POST /api/resources/sync` - Синхронизировать ресурсы из СБИС

### 3. База данных

#### Таблица `client_resources`
Хранит информацию о всех ресурсах клиентов:
- Тип ресурса (fn, evotor, atol, ofd, license, subscription, other)
- Название и серийный номер
- Даты начала и окончания
- Цена продления
- Настройка автопродления
- Связь с СБИС (sbis_resource_id, sbis_contract_id)
- Статус и метаданные

### 4. Фоновые задачи

#### `resourceMonitor.js`
- **9:00** - Проверка ресурсов с приближающимся сроком действия
- **10:00** - Автоматическое продление ресурсов

## 🔔 Уведомления

### Типы уведомлений:
- `resource_expiring` - Ресурс скоро истекает
- `resource_renewed` - Ресурс успешно продлен
- `resource_renewal_failed` - Ошибка продления

### Каналы доставки:
- ✅ В приложении (таблица notifications)
- ✅ Email (если настроен SMTP)
- ✅ Telegram (если настроен бот)

## 🚀 Использование

### 1. Синхронизация ресурсов из СБИС

```javascript
// В мобильном приложении
POST /api/resources/sync
```

### 2. Добавление ресурса вручную

```javascript
POST /api/resources
{
  "resource_type": "fn",
  "resource_name": "ФН-1",
  "serial_number": "1234567890",
  "expiry_date": "2025-12-31",
  "renewal_price": 5000,
  "auto_renewal": true
}
```

### 3. Получение ресурсов клиента

```javascript
GET /api/resources?status=active&resource_type=fn
```

## 📊 Статистика

Система автоматически:
- Отслеживает все ресурсы клиентов
- Отправляет уведомления за 60, 30 и 7 дней до окончания
- Продлевает ресурсы автоматически при наличии средств
- Создает счета в СБИС при продлении
- Синхронизирует данные со СБИС

## 🔐 Безопасность

- Все запросы требуют авторизации (JWT токен)
- Ресурсы привязаны к клиенту
- Автопродление только при достаточном балансе
- Все операции логируются

## 📝 Примечания

- Синхронизация ресурсов из СБИС требует настройки соответствующих API
- Для работы с ФН нужен доступ к API ОФД
- Для работы с лицензиями Эвотор/Атол нужны соответствующие интеграции
