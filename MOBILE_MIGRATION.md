# Миграция Mobile приложения

## 📱 Текущая ситуация

В проекте есть два мобильных приложения:

1. **Android приложение** (`app/`) - ✅ **Нативное Kotlin приложение**
2. **React Native приложение** (`mobile/`) - ❌ **Устаревшее, можно удалить**

## ✅ Рекомендация: Использовать Android приложение

### Почему Android приложение лучше:

1. **Нативный код** - быстрее и эффективнее
2. **Уже на Kotlin** - единый язык с backend
3. **Полностью готово** - все экраны реализованы
4. **Лучшая производительность** - нет JavaScript моста
5. **Проще поддерживать** - один язык для всего проекта

### Android приложение включает:

- ✅ Все экраны (Login, Register, Dashboard, Services, History, etc.)
- ✅ API интеграция через Retrofit
- ✅ JWT аутентификация
- ✅ Material Design UI
- ✅ Навигация и routing

## 🗑️ Удаление React Native приложения

React Native приложение в папке `mobile/` можно безопасно удалить:

```bash
# Удалить папку mobile
rm -rf mobile/
```

Или оставить для справки, но оно больше не используется.

## 🔄 Что делать дальше

### 1. Использовать Android приложение

Android приложение уже готово и работает с новым Kotlin backend без изменений!

### 2. Если нужен iOS

Если в будущем понадобится iOS приложение, можно:
- Создать нативное iOS приложение на Swift
- Или использовать Kotlin Multiplatform Mobile (KMM) для общего кода

### 3. Обновить документацию

Удалить упоминания React Native из README и других документов.
