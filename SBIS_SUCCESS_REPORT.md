### 2. Правильный формат запросов

Исправлены все проблемы с форматом:

#### ❌ Было (не работало):
```javascript
{
  jsonrpc: '2.0',
  method: 'СБИС.Аутентифицировать',
  params: {
    Логин: 'login',
    Пароль: 'password'
  },
  id: Date.now()
}
```

#### ✅ Стало (работает):
```javascript
{
  jsonrpc: '2.0',
  method: 'СБИС.Аутентифицировать',
  params: {
    Параметр: {  // ВАЖНО: обертка!
      Логин: 'login',
      Пароль: 'password'
    }
  },
  protocol: 2,  // ВАЖНО: версия протокола!
  id: 0
}
```

#### Хедеры:
```javascript
{
  'Host': 'online.sbis.ru',
  'Content-Type': 'application/json-rpc; charset=utf-8',
  'Accept': 'application/json-rpc',
  'X-SBISSessionID': 'session_id'
}
```

## 🔑 Ключевые решения

### Проблема 1: Методы возвращали 404
**Причина:** Неправильный формат параметров и отсутствие `protocol: 2`  
**Решение:** Добавили правильную структуру из официальной документации

### Проблема 2: OAuth не работал
**Причина:** Приложение в СБИС настроено неправильно  
**Решение:** Использовали обычную авторизацию (логин/пароль), которая работает отлично

### Проблема 3: API "Все о компаниях" требует лицензию
**Причина:** Платная подписка на SppAPI  
**Решение:** Используем CRM методы (`Контрагент.ПоИННКППКФ`), которые доступны бесплатно

## 💡 Важные заметки

### OAuth vs Login/Password

Мы **НЕ** используем OAuth, потому что:
- ❌ OAuth требует специальной настройки приложения в СБИС
- ❌ Ваше приложение не было найдено (ошибка "Приложение не найдено")
- ✅ Обычная авторизация работает для всех CRM методов
- ✅ Проще в реализации и поддержке
