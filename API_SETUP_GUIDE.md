# 🔧 Настройка API для Android приложения

## Проблема: Не могу войти в аккаунт

Если вы не можете войти в аккаунт в Android приложении, скорее всего проблема в настройке URL API сервера.

## ✅ Решение

### Шаг 1: Узнайте IP адрес вашего компьютера

**Windows:**
1. Откройте командную строку (Win + R → `cmd`)
2. Введите: `ipconfig`
3. Найдите "IPv4 Address" (например: `192.168.1.100`)

**Mac/Linux:**
1. Откройте терминал
2. Введите: `ifconfig` или `ip addr`
3. Найдите IP адрес (обычно начинается с `192.168.` или `10.`)

### Шаг 2: Убедитесь, что сервер запущен

1. Запустите ваш backend сервер (обычно `npm start` или `node server.js`)
2. Сервер должен быть доступен на порту `3000`
3. Проверьте в браузере: `http://localhost:3000/api/` (должен отвечать)

### Шаг 3: Настройте URL в Android приложении

#### Вариант A: Через настройки приложения (Рекомендуется)

1. Откройте приложение на телефоне
2. Перейдите в **Настройки** (иконка шестеренки внизу)
3. Найдите раздел **"Настройки API"**
4. Введите URL: `http://ВАШ_IP:3000/api/`
   - Например: `http://192.168.1.100:3000/api/`
5. Нажмите **"Сохранить URL"**
6. Нажмите **"Проверить подключение"** для проверки
7. Попробуйте войти снова

#### Вариант B: Через код (для разработчиков)

Откройте файл: `app/src/main/java/com/example/worldcashbox/data/api/ApiConfig.kt`

Измените:
```kotlin
private const val DEFAULT_DEVICE_URL = "http://ВАШ_IP:3000/api/"
```

Например:
```kotlin
private const val DEFAULT_DEVICE_URL = "http://192.168.1.100:3000/api/"
```

### Шаг 4: Убедитесь, что телефон и компьютер в одной сети Wi-Fi

- Телефон и компьютер должны быть подключены к одной Wi-Fi сети
- Если компьютер подключен по кабелю, убедитесь, что он в той же сети

### Шаг 5: Проверьте файрвол

**Windows:**
1. Откройте "Брандмауэр Защитника Windows"
2. Разрешите входящие подключения на порт 3000

**Mac:**
1. Системные настройки → Безопасность → Файрвол
2. Разрешите Node.js или ваш сервер

## 🔍 Диагностика проблем

### Проблема: "Не удалось подключиться к API"

**Решения:**
1. Проверьте, что сервер запущен: `http://localhost:3000/api/`
2. Проверьте IP адрес компьютера
3. Убедитесь, что телефон и компьютер в одной сети
4. Проверьте файрвол
5. Попробуйте отключить VPN на телефоне

### Проблема: "Ошибка 401" или "Неверный логин/пароль"

**Решения:**
1. Убедитесь, что используете правильные учетные данные
2. Проверьте, что сервер использует ту же базу данных, что и веб-версия
3. Попробуйте зарегистрировать новый аккаунт через приложение

### Проблема: Работает на эмуляторе, но не на реальном устройстве

**Причина:** Эмулятор использует `10.0.2.2`, а реальное устройство нужен IP компьютера

**Решение:** Настройте URL через настройки приложения (см. Шаг 3, Вариант A)

## 📱 Для эмулятора Android

Если вы используете Android эмулятор, URL должен быть:
```
http://10.0.2.2:3000/api/
```

Это специальный адрес, который эмулятор использует для доступа к localhost компьютера.

## 🌐 Для реального устройства

URL должен быть:
```
http://ВАШ_IP_АДРЕС:3000/api/
```

Например:
```
http://192.168.1.100:3000/api/
```

## ✅ Проверка

После настройки:
1. Откройте приложение
2. Перейдите в Настройки
3. Нажмите "Проверить подключение"
4. Должно появиться сообщение "✓ Подключение успешно!"
5. Попробуйте войти в аккаунт

## 💡 Полезные советы

1. **Используйте статический IP** для компьютера, чтобы IP не менялся
2. **Сохраните URL** в настройках приложения
3. **Проверяйте подключение** перед входом
4. **Используйте HTTPS** в production (требует SSL сертификат)

## 🆘 Если ничего не помогает

1. Проверьте логи сервера на наличие ошибок
2. Проверьте логи Android приложения через Logcat
3. Убедитесь, что база данных доступна для сервера
4. Проверьте, что сервер принимает запросы с вашего IP адреса
