# 🚀 Запуск бэкенда через Android Studio

## Способ 1: Через терминал Android Studio (Рекомендуется)

### Шаг 1: Откройте терминал в Android Studio

1. Внизу Android Studio нажмите на вкладку **"Terminal"**
2. Или: `View` → `Tool Windows` → `Terminal`

### Шаг 2: Перейдите в папку бэкенда

```bash
cd backend-kotlin
```

### Шаг 3: Проверьте .env файл

Убедитесь, что файл `backend-kotlin/.env` существует и содержит:

```env
PORT=3000
JWT_SECRET=MySecretKey123!
DB_HOST=localhost
DB_PORT=5432
DB_NAME=billing_db
DB_USER=postgres
DB_PASSWORD=ваш_пароль
```

### Шаг 4: Запустите бэкенд

```bash
# Windows
.\gradlew.bat run

# Linux/Mac
./gradlew run
```

### Шаг 5: Проверьте запуск

В терминале должно появиться:
```
🚀 Server running on port 3000
📊 Environment: development
✅ Database connection established
```

## Способ 2: Через конфигурацию запуска (Если проект открыт как Gradle проект)

### Шаг 1: Откройте бэкенд как отдельный проект

1. `File` → `Open`
2. Выберите папку `backend-kotlin`
3. Android Studio откроет его как отдельный проект

### Шаг 2: Создайте конфигурацию запуска

1. Вверху справа нажмите на выпадающий список конфигураций
2. Выберите `Edit Configurations...`
3. Нажмите `+` → `Application`
4. Настройте:
   - **Name:** `Backend Server`
   - **Main class:** `com.worldcashbox.ApplicationKt`
   - **Working directory:** `$PROJECT_DIR$`
   - **Use classpath of module:** `backend-kotlin.main`

### Шаг 3: Запустите

Нажмите кнопку **Run** (зеленая стрелка) или `Shift+F10`

---

## Способ 3: Через Gradle задачи

### Шаг 1: Откройте Gradle панель

1. Справа нажмите на вкладку **"Gradle"**
2. Если её нет: `View` → `Tool Windows` → `Gradle`

### Шаг 2: Найдите задачу запуска

1. Разверните: `backend-kotlin` → `Tasks` → `application`
2. Дважды кликните на `run`

---

## ⚠️ Важно!

### Бэкенд должен работать ВСЕГДА, пока вы тестируете Android приложение

- Не закрывайте терминал с бэкендом
- Оставьте его запущенным в отдельном окне терминала
- Или запустите в фоновом режиме

### Проверка работы бэкенда:

Откройте в браузере: **http://localhost:3000/health**

Должен вернуться: `{"status":"ok"}`

## 🔧 Если бэкенд не запускается

### Ошибка: "JWT_SECRET not set"

**Решение:**
1. Откройте файл `backend-kotlin/.env`
2. Добавьте строку: `JWT_SECRET=MySecretKey123!`
3. Сохраните файл
4. Перезапустите бэкенд

### Ошибка: "Database connection failed"

**Решение:**
- Убедитесь, что PostgreSQL запущен
- Проверьте пароль в `.env` файле
- Или временно закомментируйте подключение к БД (не рекомендуется)

### Ошибка: "Port 3000 already in use"

**Решение:**
1. Найдите процесс, использующий порт 3000
2. Закройте его
3. Или измените порт в `.env`: `PORT=3001`

## 💡 Совет

Создайте два окна терминала в Android Studio:
- **Терминал 1:** Запущен бэкенд (не закрывайте!)
- **Терминал 2:** Для других команд (сборка, тесты и т.д.)

Или используйте встроенный терминал Windows/командную строку для бэкенда, а Android Studio оставьте только для разработки приложения.
