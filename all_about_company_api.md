sbis.ru 2
Описание
API состоит из набора методов, вызываемых по протоколу HTTPS. Формализованный ответ сервера
возвращается в формате JSON. Перед использованием методов необходимо пройти
аутентификацию и получить SID для использования в заголовках запросов.
Тарифы
На данный момент поддерживается два тарифа базовый и расширенный. Информация, доступная
только в расширенном тарифе выделена отдельным цветом.
Порядок выполнения запроса
Для выполнения запроса впервые, нужно авторизоваться на сервисе api.sbis.ru. Для этого
необходимо отправить запрос по адресу https://api.sbis.ru/auth/service/.
Заголовки запроса:
content-type: application/json; charset=UTF-8
Тело запроса:
{
"jsonrpc": "2.0",
"method": "САП.Аутентифицировать",
"protocol": 3,
"params": {
"login": "логин*учетной*записи",
"password": "пароль"
},
"id": 1
}
, где
логин*учетной*записи и
пароль - это ваш логин и пароль соответственно на
online.sbis.ru
После успешной аутентификации надо сохранить полученный SID. (например, sid=00000000-
00000000-0000-0000000000000000;) и в будущем передавать его в Заголовок запроса к API.
Если необходимо закрыть сессию, нужно послать следующий запрос:
{
"jsonrpc": "2.0",
"method": "СБИС.Выход",
"params": {},
"id": 0
}
на адрес https://api.sbis.ru/auth/service/

Структура данных
API-интерфейс сервиса «Все о компаниях» использует формат JSON-RPC для обмена данными с
сервером api.sbis.ru посредством HTTPS-запросов/ответов. Используемая кодировка – UTF-8, метод
— POST.
Адрес запроса
https://api.sbis.ru/spp-rest-api/service/
Заголовок запроса
В заголовке передаваемого запроса обязательно указывается:
• идентификатор сессии — Cookie: sid=00000000-00000000-0000-0000000000000000,
комбинация символов(sid), полученная при авторизации
• кодировка символов — Content-Type: application/json; charset=utf-8
• данные о вашей информационной системе — User-Agent: Python-urllib/3.3
Пример:
content-type: application/json; charset=UTF-8
cookie: sid=00000000-00000000-0000-0000000000000000
user-agent: Python-urllib/3.3
Объект запроса
Объект запроса должен содержать:
• значение "jsonrpc":"2.0", указывающее на версию спецификации JSON-RPC;
• параметр "method" с названием команды;
• параметр "params" с объектом передаваемых параметров команды;
• параметр "id" со строковым значением.
Пример запроса для получения реквизитов:
{
"jsonrpc": "2.0",
"method": "SppAPI.Requisites",
"params": {
"inn": "7712040126",
"ogrn": null
},
"protocol": 4,
"id": 0
}
Объект ответа
В ответ на запрос с сервера возвращается аналогичная JSON-структура:
• значение "jsonrpc":"2.0" указывающее на версию спецификации JSON-RPC;
• параметр "id" со строковым значением, аналогично объекту запроса;
• значение объекта "result", где упаковывается ответ на команду.

SppAPI.GetRelated
SppAPI.GetRelated – возвращает список связанных компаний для выбранной постранично. На
каждой странице не более 20 элементов.
Параметр Тип Значение
requisite String ИНН или ОГРН выбранного
контрагента.
page integer Номер страницы от 1.
Возвращает массив JSON следующей структуры:
Поле Тип Описание
INN TEXT ИНН компании или ИП
Region TEXT Название региона, например «Москва»
Name TEXT Сокращенное название
Activity TEXT Вид деятельности, например «Финансовое
посредничество, не включенное в другие группировки»
State Enum Состояние организации
0 – ликвидированное
1 – действующее
2 – в состоянии реорганизации
3 – неопределено
4 – в состоянии ликвидации
DateLiquidation Date Дата ликвидации
DateRegistration Date Дата регистрации
Пример вызова в демо-режиме по компании Аэрофлот:
{"jsonrpc":"2.0","method":"SppAPI.GetRelated","params":{"requisite": "7712040126",
"page": 1},"protocol":4,"id":0}
Contractor.Find
Contractor.Find - возвращает результат полнотекстового поиска по переданной строке реквизитов.
Поиск осуществляется по следующим данным ИНН, ОГРН, Название, Адрес, телефон, e-mail, web-
сайт. В поиске участвуют только головные контрагенты, по филиалам поиск не производится.
Имеется ограничение найденного: не более 1000 элементов (как правило, этого достаточно, чтобы
найти искомого контрагента, либо надо детализировать поисковый запрос).
Параметр Тип Значение
requisites String Реквизиты искомого
контрагента.
