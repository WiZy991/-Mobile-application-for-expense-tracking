# 🎯 Интеграция СБИС CRM - Полная документация

## ✅ Статус: Работает полностью!

Реализована полная интеграция с СБИС CRM API для создания контрагентов, клиентов и сделок.

---

## 📋 Доступные API методы

### 1. Авторизация

**Endpoint:** `POST /api/sbis-proxy/auth`

**Параметры:**
```json
{
  "login": "ваш_логин",
  "password": "ваш_пароль",
  "userId": "optional_user_id"
}
```

**Ответ:**
```json
{
  "success": true,
  "onlineSession": "session_id",
  "sppSession": "spp_session_id"
}
```

---

### 2. Поиск/создание контрагента (организация)

**Endpoint:** `POST /api/sbis-proxy/crm-client-oauth`

**Описание:** Ищет контрагента по ИНН в СБИС CRM. Если не найден - создает новый.

**Параметры:**
```json
{
  "inn": "1234567890",
  "userId": "optional_user_id",
  "includeDeals": true,
  "includeDocuments": true
}
```

**Ответ:**
```json
{
  "success": true,
  "data": {
    "found": true,
    "contractor": {
      "id": 123973,
      "inn": "1234567890",
      "kpp": "",
      "name": "Контрагент 1234567890",
      "source": "sbis_crm",
      "message": "Контрагент найден/создан в СБИС CRM"
    },
    "deals": [],
    "documents": [],
    "errors": []
  }
}
```

**Используемый СБИС метод:** `Контрагент.ПоИННКППКФ`

---

### 3. Создание клиента (физическое лицо)

**Endpoint:** `POST /api/sbis-proxy/crm-create-customer`

**Описание:** Создает физическое лицо в CRM СБИС.

**Параметры:**
```json
{
  "userId": "optional_user_id",
  "surname": "Иванов",
  "name": "Иван",
  "patronymic": "Иванович",
  "gender": 0,
  "address": "г. Москва, ул. Тестовая, д. 1",
  "phone": "+7 (999) 123-45-67",
  "email": "test@example.com"
}
```

**Ответ:**
```json
{
  "success": true,
  "customerId": 142705
}
```

**Используемый СБИС метод:** `CRMClients.SaveCustomer`

---

### 4. Получение тем отношений

**Endpoint:** `POST /api/sbis-proxy/crm-get-themes`

**Описание:** Получает тему отношений по названию (нужна для создания сделок).

**Параметры:**
```json
{
  "userId": "optional_user_id",
  "themeName": "Продажи"
}
```

**Ответ:**
```json
{
  "success": true,
  "theme": {
    "d": {
      "ИдентификаторТемы": "uuid",
      "НаименованиеТемы": "Продажи",
      "Регламент": 24517561
    }
  }
}
```

**Используемый СБИС метод:** `CRMLead.getCRMThemeByName`

---

### 5. Создание сделки

**Endpoint:** `POST /api/sbis-proxy/crm-create-lead`

**Описание:** Создает сделку по контрагенту в CRM СБИС.

**Параметры:**
```json
{
  "userId": "optional_user_id",
  "clientId": 123973,
  "themeId": 24517561,
  "userConds": {
    "Источник": "Мобильное приложение",
    "Комментарий": "Тестовая сделка"
  },
  "nomenclatures": [
    {
      "code": "SERVICE-001",
      "price": 5000,
      "count": 1
    }
  ]
}
```

**Ответ:**
```json
{
  "success": true,
  "lead": {
    "d": {
      "@Документ": 201653,
      "ИдентификаторДокумента": "uuid",
      "Регламент": 24517561,
      "Клиент": { ... }
    }
  }
}
```

**Используемый СБИС метод:** `CRMLead.insertRecord`

## 🔧 Технические детали

### Формат запросов к СБИС API

Все запросы к СБИС используют правильный формат из официальной документации:

```javascript
{
  "jsonrpc": "2.0",
  "method": "Метод.API",
  "params": {
    // Параметры метода
  },
  "protocol": 2,  // ВАЖНО!
  "id": 0
}
```

### Авторизация

Используется метод `СБИС.Аутентифицировать` с оберткой `Параметр`:

```javascript
{
  "jsonrpc": "2.0",
  "method": "СБИС.Аутентифицировать",
  "params": {
    "Параметр": {  // ВАЖНО: обертка Параметр
      "Логин": "login",
      "Пароль": "password"
    }
  },
  "protocol": 2,
  "id": 0
}
```

### Хедеры запросов

```javascript
{
  "Host": "online.sbis.ru",
  "Content-Type": "application/json-rpc; charset=utf-8",
  "Accept": "application/json-rpc",
  "X-SBISSessionID": "session_id"  // После авторизации
}
```

## 🔑 Ключевые находки

### Почему не работало раньше

1. ❌ **Неправильный формат параметров** - не использовали обертку `"Параметр"` для авторизации
2. ❌ **Отсутствие protocol** - не указывали `"protocol": 2`
3. ❌ **Неправильные хедеры** - не добавляли `Host` и `Accept`
4. ❌ **Неправильная структура params** - для `Контрагент.ПоИННКППКФ` нужна двойная обертка

### Что исправили

1. ✅ **Правильный формат авторизации** - с оберткой `Параметр`
2. ✅ **Добавили protocol: 2** - версия протокола
3. ✅ **Правильные хедеры** - Host, Content-Type, Accept
4. ✅ **Правильная структура params** - двойная обертка для recordset

- **CRMLead.insertRecord** - работает ✅
- **SppAPI.Requisites (Все о компаниях)** - требует платную подписку ❌

### OAuth vs Login/Password

Мы используем **обычную авторизацию** (логин/пароль), а не OAuth, потому что:
- OAuth требует регистрации приложения в СБИС
- Обычная авторизация работает для всех методов CRM
- Проще в реализации и поддержке
