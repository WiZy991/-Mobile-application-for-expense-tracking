# Быстрое исправление ошибки поддержки

## Проблема
Ошибка `ERR_CONNECTION_REFUSED` при отправке тикета поддержки означает, что сервер не запущен или недоступен.

## Решение

### 1. Убедитесь, что сервер запущен

```bash
cd backend
npm start
```

Или если используете nodemon:
```bash
npm run dev
```

Сервер должен запуститься на порту 3000 и вывести:
```
🚀 Server running on port 3000
📊 Environment: development
```

### 2. Проверьте подключение к базе данных

Убедитесь, что:
- PostgreSQL запущен
- База данных `billing_db` создана
- Все таблицы созданы (запустите миграции если нужно)

### 3. Проверьте URL API

В файле `mobile/src/services/api.js` должен быть правильный URL:
```javascript
const API_URL = 'http://localhost:3000/api';
```

Для веб-версии может потребоваться использовать `http://127.0.0.1:3000/api` вместо `localhost`.

### 4. Проверьте, что таблицы созданы

Убедитесь, что в базе данных есть таблицы:
- `support_tickets`
- `support_messages`
- `orders`
- `staff`

Если таблиц нет, запустите инициализацию БД:
```bash
cd backend
npm run migrate
```

### 5. Проверьте CORS настройки

В `backend/src/server.js` должно быть:
```javascript
app.use(cors());
```

Это позволит веб-версии приложения подключаться к серверу.

## Тестирование

После запуска сервера проверьте:
1. Откройте в браузере: `http://localhost:3000/health`
2. Должен вернуться JSON с `status: 'ok'`
3. Попробуйте отправить тикет поддержки снова

## Альтернативное решение для веб-версии

Если вы используете веб-версию и localhost не работает, попробуйте:
1. Использовать IP адрес вместо localhost: `http://127.0.0.1:3000/api`
2. Или настроить прокси в `package.json` мобильного приложения
