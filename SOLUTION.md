# ✅ Решение проблемы с данными контрагентов

## 🔍 Проблема

При регистрации с ИНН 252700034207 приложение находило контрагента в CRM, но показывало:
- ❌ Название: "Контрагент 252700034207"
- ❌ Документов: 0
- ❌ Сделок: 0
- ❌ Адрес, телефон, email: пусто

**Причина:** У вас нет API доступа к чтению полной информации о контрагентах из СБИС CRM.

## ✅ Решение

Добавлено поле для **ручного ввода названия организации** при регистрации.

### Как это работает теперь:

1. **Пользователь вводит:**
   - ИНН: `252700034207`
   - Название: `Лим Марина Сен-Дебовна, ИП`

2. **Приложение:**
   - Пытается найти контрагента в СБИС CRM
   - Если данные не полные - использует введенное название
   - Отображает правильное название в профиле

3. **Результат:**
   ```
   ✅ Название: Лим Марина Сен-Дебовна, ИП
   ✅ ИНН: 252700034207
   ℹ️  Данные введены вручную
   ```

### 1. Запустите приложение


- Нажмите "Далее"

### 3. Введите данные организации

**ИНН:**

## 🔧 Технические детали

### Изменения в коде:

1. **Добавлен state для названия:**
   ```javascript
   const [companyName, setCompanyName] = useState('')
   ```

2. **Добавлена валидация:**
   ```javascript
   if (!companyName || companyName.trim().length < 3) {
     Alert.alert('Ошибка', 'Введите название организации')
     return false
   }
   ```

3. **Использование введенного названия:**
   ```javascript
   // Если СБИС не вернул название или оно пустое
   if (!displayName || displayName === `Контрагент ${inn}`) {
     displayName = companyName // Используем введенное пользователем
   ```

## 🚀 Преимущества решения

1. **✅ Работает всегда** - не зависит от API СБИС
2. **✅ Простое** - пользователь просто вводит название
3. **✅ Гибкое** - если API вернет данные - используем их, если нет - введенные вручную
4. **✅ Понятное** - пользователь видит что данные введены вручную
