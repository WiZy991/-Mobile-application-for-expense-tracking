# 📊 Руководство по интеграции CRM СБИС

## Что изменилось

Теперь приложение **автоматически загружает данные клиента из вашей CRM СБИС** при регистрации, а не из публичной базы ЕГРЮЛ.

### Основные изменения:

1. ✅ **Приоритет CRM**: Сначала ищем клиента в вашей CRM СБИС
2. ✅ **Автоматическая OAuth авторизация**: Используется сервисная авторизация с расширенными правами
3. ✅ **Полная информация**: Загружаются сделки, документы, контактная информация
4. ✅ **Fallback на ЕГРЮЛ**: Если клиента нет в CRM - ищем в публичной базе

---

## 🎯 Как это работает

### Процесс регистрации:

```
1. Клиент вводит ИНН
   ↓
2. Поиск в CRM СБИС (через OAuth)
   ├─→ ✅ Найден в CRM
   │   ├─ Загружаем реквизиты (ИНН, КПП, ОГРН, адрес, контакты)
   │   ├─ Загружаем список сделок
   │   ├─ Загружаем список документов
   │   └─ Показываем "✓ Данные загружены из вашей CRM СБИС"
   │
   └─→ ❌ Не найден в CRM
       ↓
       Поиск в публичной базе ЕГРЮЛ
       └─ Показываем "✓ Данные подтверждены из ЕГРЮЛ"
```

## 🔧 API Endpoints

### 1. `/api/sbis-proxy/crm-client-oauth` (НОВЫЙ)

**Поиск клиента в CRM с OAuth авторизацией**

### 2. `/api/sbis-proxy/auth-service`

**OAuth авторизация в СБИС**

Выполняет сервисную авторизацию с `app_client_id` и `app_secret` для получения расширенных прав доступа к CRM API.

#### Запрос:

```javascript
POST /api/sbis-proxy/auth-service
Content-Type: application/json

{
  "userId": "optional-user-id"
}
```

#### Ответ:

```json
{
  "success": true,
  "sessionId": "00000000-00000000-0000-0000000000000000",
  "message": "Service authentication successful"
}
```

## 📱 Использование в мобильном приложении

### Функция `sbisGetClientFromCRM`

```javascript
import { sbisGetClientFromCRM } from '../services/api';

// Поиск клиента в CRM при регистрации
const crmResult = await sbisGetClientFromCRM('7707083893');

if (crmResult.success && crmResult.data.found) {
  console.log('✅ Клиент найден в CRM!');
  console.log('Название:', crmResult.data.contractor.name);
  console.log('Сделок:', crmResult.data.deals.length);
  console.log('Документов:', crmResult.data.documents.length);
  
  // Используем данные для регистрации
  setCompanyData({
    name: crmResult.data.contractor.name,
    inn: crmResult.data.contractor.inn,
    deals: crmResult.data.deals,
    documents: crmResult.data.documents,
    fromCRM: true // Флаг, что данные из CRM
  });
} else {
  console.log('⚠️ Клиент не найден в CRM, ищем в ЕГРЮЛ...');
  // Fallback на публичную базу
}
```

## 🔑 Настройка

### 1. Учетные данные OAuth

В файле `backend/src/routes/sbisProxy.js` настроены:

```javascript
const SBIS_APP_CLIENT_ID = '5711216329531646';
const SBIS_APP_SECRET = 'LZTMTNJ5F4ZJKUMSTILYTZVW';
```

**⚠️ ВАЖНО**: Используйте ваши собственные учетные данные OAuth из личного кабинета СБИС!

### 2. Получение OAuth credentials

1. Зайдите на https://online.sbis.ru
2. Перейдите в **Настройки** → **Интеграция** → **API**
3. Создайте новое приложение или используйте существующее
4. Скопируйте `app_client_id` и `app_secret`
5. Замените значения в `sbisProxy.js`

## 🔍 Методы API CRM СБИС

Приложение пробует следующие методы для поиска клиента:

### Официальные методы (приоритет):

1. **`Навигация.Контрагент.Список`** - список контрагентов с фильтрацией
2. **`Навигация.Контрагент.Прочитать`** - чтение контрагента по ИНН
3. **`CRMLead.getList`** - список сделок в CRM
4. **`CRMLead.read`** - чтение конкретной сделки

### Старые методы (fallback):

1. `Контрагент.Список`
2. `Контрагент.Прочитать`
3. `Справочник.Контрагенты`
4. `Реестр.СписокЗаписей`

## ❓ Частые вопросы

### Q: Почему клиент не находится в CRM?

**A:** Возможные причины:

1. ❌ Клиент действительно отсутствует в вашей CRM СБИС
2. ❌ Недостаточно прав у OAuth токена
3. ❌ Неправильные `app_client_id` / `app_secret`
4. ❌ Не подключен тариф "CRM" в СБИС

**Решение**: Проверьте:
- Наличие клиента в веб-интерфейсе СБИС
- Права доступа OAuth приложения
- Активные тарифы в личном кабинете

### Q: Получаю ошибку 403 Forbidden

**A:** Это означает недостаточно прав доступа.

**Решение**:
1. Убедитесь, что подключен тариф "CRM"
2. Проверьте права OAuth приложения в настройках СБИС
3. Убедитесь, что используете правильные credentials

### Q: Получаю ошибку 404 Not Found

**A:** Метод не существует или недоступен.

**Решение**:
1. Запустите диагностику: `/api/sbis-proxy/diagnostics`
2. Проверьте доступные методы в личном кабинете СБИС
3. Возможно, метод называется по-другому в вашей версии СБИС

### Q: Как узнать, какие методы доступны?

**A:** Используйте эндпоинт диагностики:

```bash
curl -X POST http://localhost:3000/api/sbis-proxy/diagnostics \
  -H "Content-Type: application/json" \
  -d '{"userId": "test"}'
```

Или запустите тестовый скрипт (см. раздел Тестирование).
