# 🔍 Проверка статуса бэкенда

## Проблема
Android приложение не может подключиться к `10.0.2.2:3000` - бэкенд не отвечает.

## Проверка в Android Studio

### 1. Проверьте, что бэкенд действительно запущен

**В Android Studio:**
1. Посмотрите внизу на вкладку **"Run"** или **"Build"**
2. Должна быть запущена конфигурация с бэкендом
3. В консоли должны быть логи типа:
   ```
   🚀 Server running on port 3000
   ✅ Database connection established
   ```

### 2. Если бэкенд не запущен или падает:

**Проверьте логи бэкенда:**
- Внизу Android Studio откройте вкладку с логами бэкенда
- Ищите ошибки типа:
  - `JWT_SECRET not set`
  - `Database connection failed`
  - `Port 3000 already in use`

### 3. Проверьте конфигурацию запуска:

1. Вверху справа найдите выпадающий список конфигураций
2. Убедитесь, что выбрана правильная конфигурация для бэкенда
3. Если конфигурации нет - создайте её (см. инструкцию выше)

### 4. Проверьте .env файл:

Откройте `backend-kotlin/.env` и убедитесь, что там есть:
```env
PORT=3000
JWT_SECRET=MySecretKey123!
DB_PASSWORD=ваш_пароль
```

### 5. Перезапустите бэкенд:

1. Остановите бэкенд (красная кнопка Stop)
2. Запустите снова (зеленая кнопка Run)

---

## Быстрая проверка через терминал Android Studio

В терминале Android Studio выполните:

```bash
netstat -an | findstr ":3000"
```

Если ничего не выводится - бэкенд не запущен.

---

## Если бэкенд падает при запуске:

**Пришлите логи из консоли бэкенда** - там будет точная ошибка.

Частые ошибки:
- ❌ `JWT_SECRET not set` → Добавьте в `.env`: `JWT_SECRET=MySecretKey123!`
- ❌ `Database connection failed` → Проверьте пароль БД в `.env`
- ❌ `Port 3000 already in use` → Измените порт в `.env`: `PORT=3001`
