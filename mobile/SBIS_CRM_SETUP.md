# 🎯 Настройка СБИС CRM интеграции в мобильном приложении

## ✅ Готово к использованию!

Интеграция СБИС CRM уже полностью настроена в мобильном приложении!

---

## 🔧 Как это работает

### При регистрации клиента:

1. **Клиент вводит ИНН** на экране регистрации
2. **Приложение авторизуется** в СБИС с credentials из `sbisConfig.js`
3. **Ищет клиента в вашей CRM** по ИНН
4. **Загружает данные:**
   - Контактную информацию (название, адрес, телефон, email)
   - Список сделок с суммами
   - Документы и счета
   - Историю сотрудничества

5. **Отображает всё это** на экране успешной регистрации

---

## ⚙️ Настройка

### 1. Проверьте credentials в `mobile/src/config/sbisConfig.js`:

```javascript
export const SBIS_CONFIG = {
  login: 'tenditnika',      // ✅ Ваш логин СБИС
  password: 'Tenditnik1!',  // ✅ Ваш пароль
  sellerINN: '2543082240',  // ✅ ИНН вашей организации
  sellerKPP: '253601001',   // ✅ КПП
  sellerName: 'WorldCashBox', // ✅ Название
  demoMode: false,          // ✅ Отключен demo-режим
}
```

**ВАЖНО:** Убедитесь, что логин и пароль правильные!

## 📱 Как использовать

### Регистрация нового клиента:

1. Откройте приложение и нажмите "Регистрация"
2. Введите данные пользователя (телефон, email, пароль)
3. На следующем экране введите **ИНН организации клиента**
4. Нажмите "Найти компанию"
5. Приложение автоматически:
   - ✅ Авторизуется в СБИС
   - ✅ Найдет клиента в CRM
   - ✅ Загрузит все данные
   - ✅ Покажет сделки и документы

## 🔄 Логика работы

### Приоритет источников данных:

1. **Сначала** - ваша CRM СБИС (метод `Контрагент.ПоИННКППКФ`)
   - Если найдено ✅ - используем данные из CRM
   - Показываем бейдж "Данные загружены из вашей CRM"

2. **Потом** - публичная база ЕГРЮЛ (API "Все о компаниях")
   - Если не найдено в CRM
   - Показываем бейдж "Данные подтверждены из ЕГРЮЛ"

3. **В конце** - базовая информация
   - Если нигде не найдено
   - Показываем сообщение о настройке интеграции

## ⚠️ Типичные ошибки

### 1. "Ошибка подключения к СБИС"

**Причина:** Неверные credentials в `sbisConfig.js`

**Решение:**
- Проверьте логин и пароль
- Убедитесь, что аккаунт не заблокирован
- Попробуйте авторизоваться на https://online.sbis.ru вручную

---

### 2. "Клиент не найден в CRM"

**Причина:** Контрагента с таким ИНН нет в вашей CRM

**Решение:**
- Это нормально! Приложение попробует найти в ЕГРЮЛ
- Или создайте контрагента в СБИС вручную
- После регистрации контрагент будет автоматически создан в CRM

---

### 3. Backend недоступен

**Причина:** Backend не запущен или неверный URL

**Решение:**
1. Запустите backend: `cd backend && npm start`
2. Проверьте `mobile/app.json`:
   ```json
   "extra": {
     "apiUrl": "http://YOUR_IP:3000/api"
   }
   ```
3. Замените `YOUR_IP` на реальный IP компьютера

## 📝 Код интеграции

### Основные файлы:

1. **`mobile/src/screens/RegisterScreen.js`** - UI регистрации
   - Функция `simulateSearch()` - поиск клиента
   - Авторизация в СБИС
   - Вызов CRM API
   - Отображение результатов

2. **`mobile/src/services/api.js`** - API клиент
   - `sbisAuth()` - авторизация
   - `sbisGetClientFromCRM()` - поиск в CRM
   - Обработка ошибок

3. **`mobile/src/config/sbisConfig.js`** - Конфигурация
   - Credentials
   - Настройки API

### Пример вызова:

```javascript
// Авторизация
const authResult = await sbisAuth(
  SBIS_CONFIG.login, 
  SBIS_CONFIG.password
);

if (authResult.success) {
  // Поиск в CRM
  const crmResult = await sbisGetClientFromCRM(inn);
  
  if (crmResult.success && crmResult.data?.found) {
    // Данные найдены!
    const contractor = crmResult.data.contractor;
    const deals = crmResult.data.deals;
    const documents = crmResult.data.documents;
  }
}
```
