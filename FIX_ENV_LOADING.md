# ✅ Исправлена загрузка .env файла

## Что было исправлено:

1. ✅ Добавлена библиотека `dotenv-kotlin` для загрузки `.env` файла
2. ✅ Создана утилита `EnvUtils` для централизованного чтения переменных окружения
3. ✅ Все места, где использовался `System.getenv()`, теперь используют `EnvUtils`, который автоматически читает из `.env` файла

## Что нужно сделать:

### 1. Пересоберите проект:

```bash
cd backend-kotlin
./gradlew build
```

### 2. Убедитесь, что файл `.env` существует в папке `backend-kotlin`:

Файл должен содержать:
```env
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=billing_db
DB_USER=postgres
DB_PASSWORD=ваш_пароль_от_postgresql
JWT_SECRET=ваш_секретный_ключ
```

### 3. Запустите бэкенд:

```bash
cd backend-kotlin
./gradlew run
```

Теперь бэкенд должен автоматически загружать переменные из `.env` файла!

## Проверка:

После запуска вы должны увидеть:
```
🚀 Server running on port 3000
📊 Environment: development
```

Если видите ошибку о `JWT_SECRET`, значит `.env` файл не загружается. Проверьте:
- Файл `.env` находится в папке `backend-kotlin` (не в корне проекта)
- В файле нет лишних пробелов вокруг `=`
- Все значения указаны без кавычек
